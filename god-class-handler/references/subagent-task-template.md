# Subagent Task Template

将以下模板发给subagent，并替换占位符：

```md
你负责重构一个单一功能单元，禁止越界修改。
本任务是执行型任务，必须提交实际代码迁移结果，不可只提供分析建议。

## 目标
- 目标类: <ClassName>
- 功能单元: <UnitName>
- 关联方法: <MethodA>, <MethodB>
- 目标文件: <Path>

## 约束
- 保持行为一致，不改变业务语义。
- 不修改未授权功能单元。
- 若需改公共 API，先给兼容方案再修改。
- 本任务仅迁移 private 方法，非 private 方法禁止迁移。
- 本任务仅做物理拆分，不做逻辑重写。

## 必做事项
1. 将该单元中的 private 方法迁移到 <NewModuleOrClass>。
2. 更新原调用方式为新模块调用，不遗漏。
3. 输出方法签名前后对照（名称、参数、返回值、可见性、异常约定）。
4. 输出迁移成功证据：
   - 新实现位置（文件 + 方法名）
   - 原类旧实现处理结果（删除/委托/桥接）
   - 调用点替换结果
   - 实现完整性证明（非空实现、无 TODO/pass/未实现异常占位）
5. 删除可确认无用的旧私有辅助方法（仅限本单元）。
6. 若用户确认执行测试，补充或更新最小必要测试。
7. 执行构建；仅在用户确认时执行测试并汇报结果。
8. 禁止提交空实现或占位实现（TODO/pass/仅抛未实现异常）。

## 输出要求
1. 修改文件列表。
2. 迁移的方法清单（旧位置 -> 新位置）。
3. 调用点替换清单（旧调用 -> 新调用）。
4. 方法签名对照清单（迁移前 -> 迁移后）。
5. 迁移成功证据清单（逐方法）。
6. 实现完整性检查清单（逐方法）。
7. 删除项清单（方法/常量/字段）。
8. 风险与需主 agent 收尾事项。
9. 若未完成迁移，明确阻塞原因与缺失项（禁止宣称已完成）。
```
